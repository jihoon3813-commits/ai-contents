# 멀티플랫폼 AI 콘텐츠 제작 서비스

## MVP 기능명세서 및 데이터베이스 설계서

문서 버전: 1.0
개발 환경: Google Antigravity
서비스 형태: 반응형 웹 SaaS
주 사용 언어: 한국어
기본 시간대: Asia/Seoul

---

# 1. 프로젝트 정의

## 1.1 프로젝트 목적

사용자가 콘텐츠의 목적, 플랫폼, 카테고리, 주제, 키워드, 글자 수, 이미지 수, 문체 등을 설정하면 다음 결과물을 생성하는 웹 서비스를 개발한다.

1. 플랫폼별 콘텐츠 초안
2. SEO 제목과 메타데이터
3. 이미지 구성안과 이미지 프롬프트
4. 해시태그와 CTA
5. SEO·가독성·신뢰도 점수
6. HTML 또는 일반 텍스트 내보내기
7. 워드프레스와 Blogger 초안 발행
8. 프로젝트와 콘텐츠 버전 관리

---

## 1.2 핵심 제품 원칙

### 원칙 1. 플랫폼별로 별도 생성한다

하나의 글을 줄이거나 늘려서 재사용하지 않는다.

동일 주제라도 플랫폼에 따라 다음 항목을 다르게 생성한다.

* 제목
* 글 구조
* 문단 길이
* 키워드 사용 방식
* 이미지 수와 위치
* CTA
* 해시태그
* 메타데이터
* 콘텐츠 길이
* 말투

### 원칙 2. 사용자 제공 사실과 AI 제안을 구분한다

AI가 만들어낸 경험을 실제 경험처럼 작성하지 않는다.

각 정보는 내부적으로 다음 유형을 가진다.

* USER_FACT: 사용자가 직접 제공한 사실
* SOURCE_FACT: 참고 자료에서 확인된 사실
* GENERAL_KNOWLEDGE: 일반적인 설명
* AI_SUGGESTION: AI가 제안한 표현
* NEEDS_VERIFICATION: 확인이 필요한 정보

### 원칙 3. 자동 발행보다 검토를 우선한다

기본 생성 상태는 항상 `검토 필요`로 저장한다.

사용자가 승인한 후에만 다음 작업이 가능하다.

* 복사
* 파일 내보내기
* 외부 플랫폼 초안 전송
* 예약 발행
* 즉시 발행

### 원칙 4. 저품질 대량 생성 방지

다음 조건에서는 경고를 표시한다.

* 기존 콘텐츠와 주제가 지나치게 유사함
* 실제 경험 없이 후기형 콘텐츠를 선택함
* 출처 없는 수치가 포함됨
* 키워드가 지나치게 반복됨
* 글자 수를 채우기 위한 반복 문장이 있음
* 동일한 콘텐츠를 여러 플랫폼에 그대로 사용하려 함

---

## 4.2 권한 원칙

모든 핵심 테이블에 `workspace_id`를 둔다.

사용자는 자신이 소속된 워크스페이스 데이터만 접근할 수 있다.

Supabase RLS 정책 예시:

```sql
workspace_id IN (
  SELECT workspace_id
  FROM workspace_members
  WHERE user_id = auth.uid()
)
```

외부 플랫폼 액세스 토큰은 브라우저에 직접 노출하지 않는다.

## BRD-003 브랜드 보이스 설정

### 슬라이더

각 항목은 1~5 정수로 저장한다.

* formal_level: 캐주얼 ↔ 공식적
* sentence_length: 짧게 ↔ 길게
* expertise_level: 쉽게 ↔ 전문적으로
* emotional_level: 담백하게 ↔ 감성적으로
* sales_level: 정보 중심 ↔ 판매 중심
* humor_level: 진지하게 ↔ 유쾌하게
* emoji_level: 없음 ↔ 많음
* question_level: 질문 적게 ↔ 질문 많이

### 추가 필드

* 기본 존댓말 여부
* 자주 사용하는 표현
* 사용 금지 표현
* 브랜드 고유 용어
* CTA 스타일
* 도입부 선호 방식
* 마무리 선호 방식

## SEO-002 점수

각 점수는 0~100이다.

```text
intent_score
quality_score
readability_score
technical_seo_score
trust_score
platform_fit_score
overall_score
```

전체 점수 계산:

```text
overall_score =
intent_score × 0.25
+ quality_score × 0.20
+ readability_score × 0.15
+ technical_seo_score × 0.15
+ trust_score × 0.15
+ platform_fit_score × 0.10
```

인스타그램은 `technical_seo_score` 비중을 줄이고 `platform_fit_score`를 높인다.

# 23. 데이터베이스 설계

## 23.1 공통 규칙

모든 기본키는 UUID를 사용한다.

공통 필드:

```sql
id uuid primary key default gen_random_uuid(),
created_at timestamptz not null default now(),
updated_at timestamptz not null default now()
```

필요 테이블에는 다음 필드를 추가한다.

```sql
workspace_id uuid not null,
created_by uuid,
updated_by uuid,
deleted_at timestamptz
```

JSONB는 유동적인 플랫폼 설정과 AI 결과에만 사용한다. 검색과 관계 설정에 필요한 값은 정규 컬럼으로 저장한다.

# 38. 보안 요구사항

* 파일 크기 제한
* Rate Limit 적용
* 입력값 길이 제한
* HTML 정화
* XSS 차단
* CSRF 고려
* SQL 직접 조합 금지
* 민감 데이터 로그 제외

# 45. MVP 최종 완료 조건

다음 시나리오가 모두 작동하면 MVP 완료로 본다.

1. 사용자가 회원가입한다.
2. 브랜드를 생성한다.
3. 브랜드 말투를 설정한다.
4. 네이버 블로그·워드프레스·인스타그램을 선택한다.
5. 주제와 키워드를 입력한다.
6. 글자 수와 이미지 수를 입력한다.
7. 실제 경험 정보를 입력한다.
8. AI가 공통 개요를 생성한다.
9. 사용자가 개요를 수정하고 승인한다.
10. 플랫폼별로 서로 다른 콘텐츠가 생성된다.
11. 이미지 위치와 이미지 프롬프트가 생성된다.
12. SEO·가독성·신뢰도 점수가 표시된다.
13. 사용자가 문장을 선택해 다시 작성할 수 있다.
14. 수정 내용이 자동 저장된다.
15. 워드프레스 콘텐츠를 초안으로 전송할 수 있다.
16. 네이버 블로그 콘텐츠를 문단별로 복사할 수 있다.
17. 인스타그램 캐러셀 문구를 다운로드할 수 있다.
18. 이전 버전으로 복원할 수 있다.
19. 사용자별 사용량이 기록된다.
20. 다른 사용자의 데이터에 접근할 수 없다.

# 46. 안티그래비티 작업 지시 방식

전체 프로젝트를 한 번에 구현하지 말고 다음 형식으로 단계별 지시한다.

## 작업 요청 형식

```text
현재 작업 범위:
Phase 1 인증과 워크스페이스 기능만 구현한다.

기술 조건:
- Next.js App Router
- TypeScript
- Supabase Auth
- Supabase PostgreSQL
- Tailwind CSS
- shadcn/ui

구현 항목:
1. 이메일 회원가입
2. 이메일 로그인
3. Google 로그인
4. 워크스페이스 자동 생성
5. workspace_members 자동 생성
6. 보호된 대시보드
7. 로그아웃

완료 조건:
- 신규 가입 시 profiles, workspaces, workspace_members가 생성된다.
- 로그인하지 않은 사용자는 /login으로 이동한다.
- 사용자는 본인 워크스페이스만 조회할 수 있다.
- Playwright E2E 테스트를 작성하고 실행한다.
- TypeScript와 lint 오류가 없어야 한다.
- 구현 후 변경 파일과 테스트 결과를 보고한다.

제외 범위:
- 브랜드
- 콘텐츠 생성
- AI 연동
- 결제
- 플랫폼 연동
```

## 매 단계 공통 지시

```text
작업 전에 기존 코드 구조와 데이터베이스 스키마를 확인한다.
필요한 변경사항을 먼저 계획으로 정리한다.
기존 기능을 삭제하거나 임의 변경하지 않는다.
환경변수와 비밀키를 코드에 하드코딩하지 않는다.
모든 입력값을 Zod로 검증한다.
서버 오류와 사용자 오류를 구분한다.
완료 후 lint, typecheck, unit test, E2E test를 실행한다.
브라우저에서 핵심 사용자 흐름을 직접 검증한다.
발견한 오류를 수정한 후 최종 결과를 보고한다.
```
